# 接口文档

## 通用约定

- 基础路径：`/api`
- 鉴权：除登录外，所有接口需在 Header 携带 `Authorization: Bearer <token>`
- 统一响应格式：
```json
{ "code": 0, "message": "success", "data": { } }
```
`code=0` 成功，非 0 为错误（HTTP 状态码同步：401 未登录 / 403 无权限 / 404 不存在 / 400 参数错误 / 500 服务异常）。
- 角色：`admin`（管理员，全部权限）/ `tech`（技术员，仅自己的常规项目）。

## 2. 人员账号 `/api/users`（仅管理员）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 列表，query `keyword,role` |

## 3. 项目 `/api/projects`

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/` | 登录 | 列表分页。管理员支持全量筛选；技术员仅返回分配给自己的常规项目 |
| GET | `/:id` | 登录 | 详情（含附件、进度记录）。技术员仅可看自己的常规项目 |
| POST | `/` | 管理员 | 创建项目 |
| PUT | `/:id` | 管理员 | 编辑项目 |
| PATCH | `/:id/assign` | 管理员 | 分配技术员 body `{tech_id}`（仅常规项目） |
| PATCH | `/:id/progress` | 登录 | 进度流转 body `{to_status,note,source_uploaded}` |
| DELETE | `/:id` | 管理员 | 软删除 |
| POST | `/batch-delete` | 管理员 | 批量软删除 body `{ids:[]}` |
| GET | `/export/excel` | 管理员 | 导出 Excel（复用列表筛选参数） |

**列表筛选参数**（GET `/`）：`keyword`（模糊：项目名/客户/电话/描述/备注/技术员）、`project_type`(`1`常规/`2`历史维护完结，兼容 `normal/maintenance`)、`status`、`tech_id`、`settled`、`server_first_push`、`server_owner`、`tag_id`、`start_date/end_date`（创建时间段）、`finish_start/finish_end`（完工区间）、`page/pageSize`。

**项目字段**：
- 必填：`name,description,requirement,duration`
- 选填客户字段：`customer_name,customer_phone`
- 类型：`project_type` = `1`（常规开发项目，可分配技术员、5 阶段流转）/ `2`（历史维护完结项目，强制无技术员、状态固定 completed、技术员不可见）；接口兼容旧参数 `normal/maintenance`
- 选填：`start_time,total_reward,first_payment,mid_payment,final_payment,project_cost,tech_fee,server_first_push,server_owner,server_start_date,server_buy_date,server_expire_date,maintenance_amount,remark,actual_finish_time,settled,tag_ids[]`
- 计算字段（返回）：`income=首+中+尾`，`profit=income-成本-技术费用`（保留 2 位）

**进度流转状态**：`pending→stage1(接单)→stage2(初版展示)→stage3(终版修改)→stage4(完成源码上传)→stage5(提交付款申请)→completed`。只能顺序 +1。流转到 `stage5` 必须传 `source_uploaded`(0/1)，之后项目 `locked=1` 锁定。

## 4. 附件 `/api/files`

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| POST | `/:projectId` | 管理员 | 多文件上传（form-data 字段 `files`），单文件≤200MB，白名单扩展名 |
| GET | `/:fileId/download` | 登录 | 下载。技术员仅可下载自己常规项目的附件 |
| DELETE | `/:fileId` | 管理员 | 删除附件 |

## 5. 数据大屏 `/api/dashboard`（仅管理员）

GET `/` 默认返回全量统计（常规 + 历史维护完结项目合并）：`cards`（核心指标，含 `normalProjects/historyProjects`）、`statusDist`、`typeDist`、`typeBoard`、`monthRevenue`、`monthProfit`、`techStat`、`finishTrend`、`fundRing`、`serverPush`、`serverOwner`。

支持 query：`project_type=1/2`（兼容 `normal/maintenance`）单独查看常规或历史项目；不传则合并统计。

## 新旧项目隔离与合并规则

- 管理员视图：项目列表、台账、大屏、利润、逾期、完工统计默认合并 `project_type=1` 和 `project_type=2`。
- 技术员视图：项目列表、详情、附件下载、进度流转、个人业绩仅允许 `project_type=1 AND tech_id=当前用户`。
- 历史维护完结项目：创建/编辑时强制 `tech_id=null`、`status=completed`，禁止分配技术员，禁止进度流转，附件仅管理员可查看下载。
- 利润公式统一：`first_payment + mid_payment + final_payment - project_cost - tech_fee`，保留 2 位小数。
